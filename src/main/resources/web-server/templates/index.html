Overview

  • This demo originally defined a simple web-application for accessing accounts data.
  • All the account information is fetched via a RESTful interface to a Accounts microservice.
  • The original microservices code contained only three Spring Boot microservices:
    • A Web Front-End service (WebServer.java)
    • An "Accounts" REST microservice (AccountsServer.java)
    • A Eureka-based registration microservice (RegistrationServer.java)

Roberto's "Products" REST Microservice Addition

  • Roberto added a fourth backing "Products" REST microservice (ProductsService.java) on port 4444 in its entirety (controller, data model) and its JPA/Hibernate data store.
    • Products by number (REST API) : http://localhost:4444/products/123456020
    • Products by name (REST API) : http://localhost:4444/products/byname/microserv
    • Products search (REST API) : http://localhost:4444/products/search

Roberto's Front-End Additions

  • Roberto added various Product-based functionalities to the front end microservice on port 3333.
    • Products by number (REST passthrough) : http://localhost:3333/products/123456020
    • Products by name (REST passthrough) : http://localhost:3333/products/byname/microserv
    • Products search (form to REST passthrough) : http://localhost:3333/products/search
  • The web microservice's pages delegate REST queries to the Products microservice and wrap the results in a Spring ThymeLeaf based result HTML.
